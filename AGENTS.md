# AGENTS.md

## Cursor Cloud specific instructions

### Project Overview

ATProto Community Notes — an open community notes service for the AT Protocol (Bluesky) ecosystem. Monorepo with pnpm workspaces: `packages/notes` (core library), `packages/dev-infra` (Docker helpers for Postgres/Redis), `packages/dev-env` (process-compose dev orchestrator), `services/notes` (production entrypoint).

### Key Commands

- **Install deps**: `pnpm install --frozen-lockfile`
- **Codegen** (generates lexicon TypeScript from JSON schemas — required before first build): `cd packages/notes && pnpm exec lex gen-server --yes ./src/lexicon ./src/lexicon/org/opencommunitynotes/*`
- **Build**: `pnpm build`
- **Lint**: `ESLINT_USE_FLAT_CONFIG=false pnpm lint` (the repo uses `.eslintrc` with ESLint 9, so the flat-config flag is required)
- **Tests**: `cd packages/notes && ../dev-infra/with-test-redis-and-db.sh node --test --import=tsx tests/*.test.ts` (or `just test` from the repo root — note the justfile uses `zsh` so `zsh` must be installed)
- **Format**: `pnpm format`

### Gotchas

1. **Lexicon codegen must run before the first build.** The `packages/notes/src/lexicon/` directory only contains JSON schemas in the repo. The TypeScript files (`index.ts`, `lexicons.ts`, `types/`) are generated by `lex gen-server`. Without this step, `pnpm build` will fail with TS2307 errors about missing `../lexicon` modules.

2. **ESLint requires `ESLINT_USE_FLAT_CONFIG=false`.** The project has `.eslintrc` (legacy format) but depends on ESLint 9 which defaults to flat config. Always set `ESLINT_USE_FLAT_CONFIG=false` when running ESLint.

3. **`dev.env` has a hardcoded `LOG_DESTINATION` path** pointing to the original developer's machine. For Cloud Agent use, override it: `LOG_DESTINATION=/workspace/dev.log` when starting the dev environment.

4. **The justfile uses `zsh`** (`set shell := ["zsh", "-o", "pipefail", "-c"]`). Ensure `zsh` is installed to use `just` commands.

5. **Tests and the dev-env require Docker** for ephemeral Postgres (port 5433) and Redis (port 6380) containers. The `with-test-redis-and-db.sh` helper manages container lifecycle automatically.

6. **Starting the full dev environment** (all AT Protocol services + Community Notes): run directly rather than through process-compose:
   ```bash
   cd packages/notes && \
     LOG_DESTINATION=/workspace/dev.log NODE_ENV=development LOG_ENABLED=true LOG_LEVEL=debug \
     ../dev-infra/with-test-redis-and-db.sh node --enable-source-maps dist/dev-env/bin.js
   ```
   This starts: DID PLC (2582), PDS (2583), Bsky Appview (2584), Ozone (2587), Community Notes (2595), Internal API (2596), Test Labeler (2597), Introspection (2581). Mock data is auto-generated.

7. **Test credentials** for the dev-env: user `alice.test` / password `hunter2` on PDS at `localhost:2583`.

8. **Pre-existing lint errors exist** in the repository (formatting/prettier issues, unused vars). These are not caused by Cloud Agent changes.
