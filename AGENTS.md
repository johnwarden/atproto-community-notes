# AGENTS.md

## Cursor Cloud specific instructions

### Project Overview

ATProto Community Notes — an open community notes service for the AT Protocol (Bluesky) ecosystem. Monorepo with pnpm workspaces: `packages/notes` (core library), `packages/dev-infra` (Docker helpers for Postgres/Redis), `packages/dev-env` (process-compose dev orchestrator), `services/notes` (production entrypoint).

### Environment Setup with Devbox

All system dependencies are declared in `devbox.json`. Use [devbox](https://www.jetify.com/devbox) to provision them (Node.js 22, pnpm 10.14.0, just, process-compose, cmake, pkg-config, sqlite, clang, python3, etc.):

```bash
# Install devbox (one-time)
curl -fsSL https://get.jetify.com/devbox | bash -s -- -f

# Install Nix + all packages declared in devbox.json
devbox install

# Run any command inside the devbox environment
devbox run -- <command>
```

The devbox init_hook automatically runs `pnpm install --frozen-lockfile`.

**Docker is also required** for test Postgres (port 5433) and Redis (port 6380) containers. Docker must be installed and running separately — devbox does not manage it.

**Nix daemon**: devbox requires the Nix daemon to be running. If you see "cannot connect to socket" errors, start it with `sudo /nix/var/nix/profiles/default/bin/nix-daemon &` and ensure the current user is in `/etc/nix/nix.conf` as a trusted user.

### Key Commands

All commands below should be run via `devbox run --` to ensure the correct tool versions:

- **Install deps**: `devbox run -- pnpm install --frozen-lockfile`
- **Codegen** (required before first build — generates lexicon TypeScript from JSON schemas): `cd packages/notes && devbox run -- pnpm exec lex gen-server --yes ./src/lexicon ./src/lexicon/org/opencommunitynotes/*`
- **Build**: `devbox run -- pnpm build`
- **Lint**: `devbox run -- env ESLINT_USE_FLAT_CONFIG=false pnpm lint`
- **Tests**: `devbox run -- just test` (or run specific test: `devbox run -- just test <filename>`)
- **Format**: `devbox run -- pnpm format`
- **Typecheck**: `devbox run -- just typecheck`

### Starting the Dev Environment

```bash
devbox run -- just start   # Start all services (waits until ready)
devbox run -- just stop    # Stop gracefully
devbox run -- just status  # Show service URLs and status
devbox run -- just health  # Health check all services
```

`just start` uses process-compose to orchestrate Docker (Postgres + Redis) and the full AT Protocol test network. When ready, the following services are available:

| Introspection | 2581 | Dev-env metadata (service URLs, DB paths, mock data status) |

| Ozone | 2587 | AT Protocol moderation |

### Gotchas

1. **Lexicon codegen must run before the first build.** The `packages/notes/src/lexicon/` directory only contains JSON schemas in the repo. The TypeScript files (`index.ts`, `lexicons.ts`, `types/`) are generated by `lex gen-server`. Without this step, `pnpm build` will fail with TS2307 errors.

2. **ESLint requires `ESLINT_USE_FLAT_CONFIG=false`.** The project has `.eslintrc` (legacy format) but depends on ESLint 9 which defaults to flat config.

3. **The justfile uses `zsh`** (`set shell := ["zsh", "-o", "pipefail", "-c"]`). Devbox provides it, but if running outside devbox, ensure `zsh` is installed.

4. **Pre-existing lint errors exist** in the repository (formatting/prettier issues, unused vars). These are not caused by Cloud Agent changes.

### For Frontend Agents

If you are developing a frontend that consumes this backend, you can start the backend services from this repo:

1. **Clone this repo** to a known path (e.g., `/workspace/backend`)
2. **Install devbox** and run `devbox install` in the backend repo root
3. **Build**: `devbox run -- pnpm build` (codegen + build are handled by the update script)
4. **Start**: `devbox run -- just start`
5. **Wait** for `curl -s http://localhost:2581 | jq '.mockSetup.complete'` to return `true`
6. **Discover services**: `curl -s http://localhost:2581 | jq .` returns all service URLs
7. **Authenticate**: `curl -s -X POST http://localhost:2583/xrpc/com.atproto.server.createSession -H "Content-Type: application/json" -d '{"identifier": "alice.test", "password": "hunter2"}' | jq '.accessJwt'`

The Community Notes API is at `http://localhost:2595`. Key endpoints:
- `GET /health` — health check
- `GET /xrpc/org.opencommunitynotes.getConfig` — service configuration
- `GET /xrpc/org.opencommunitynotes.getProposals?uris=<uri>` — retrieve proposals
- `POST /xrpc/org.opencommunitynotes.propose` — create a proposal (requires auth)
- `POST /xrpc/org.opencommunitynotes.vote` — vote on a proposal (requires auth)
- `GET /xrpc/app.bsky.feed.getFeedSkeleton?feed=<feed-uri>` — feed skeleton
- `GET /xrpc/app.bsky.feed.describeFeedGenerator` — feed generator discovery

When done: `devbox run -- just stop`
